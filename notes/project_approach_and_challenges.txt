## Challenges

### Experimental API Status

The core tool, the Prompt API (window.ai.languageModel) that exposes Gemini Nano, is currently experimental (Source 1.5, 3.7). This means the API surface, required flags, and stability are subject to change without notice.

### Hardware Requirements

Gemini Nano is an on-device model, which requires specific user hardware to run: Windows 10/11, macOS 13+, Linux, or ChromeOS (Chromebook Plus), and often requires at least 22 GB of free disk space and a GPU with 4 GB VRAM (Source 3.6, 3.7). The extension will simply not work for users whose devices do not meet these specs.

### Model Download & Latency

The model needs to be downloaded to the user's device upon first use (Source 2.4, 3.7). While this is automatic, it takes time and requires ample storage. Even once downloaded, on-device inference latency will be slower than instantaneous JavaScript processing, which impacts the user experience when organizing hundreds of bookmarks.

### Chrome Bookmarks API

You must learn and correctly implement the asynchronous Chrome Extension chrome.bookmarks API (not found in search, but a standard extension requirement). This involves writing code to read the tree, delete old folders/links, and create new folders/links based on the AI's classification.
